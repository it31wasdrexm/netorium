# Руководство по тестированию Netorium (Контроллер + Агент)

Это простое пошаговое руководство поможет вам настроить **Контроллер** на одном компьютере (например, под управлением Linux, macOS или Windows) и подключить к нему **Агент** с другого компьютера (под управлением Windows). После этого вы сможете заблокировать сайты/игры и ограничить скорость интернета.

---

## 🛠️ Шаг 1. Настройка Контроллера (на Компьютере А — "Админский ПК")

Этот компьютер будет сервером управления. Это может быть любой ПК (Linux, macOS или Windows), на котором установлен Netorium.

### 1. Инициализация и запуск контроллера
Откройте терминал на Компьютере А и выполните следующие команды:

```bash
# Инициализируем конфигурацию
netorium config init

# Инициализируем базу данных контроллера
netorium controller init --host 0.0.0.0 --port 8765

# Запускаем контроллер на всех сетевых интерфейсах (чтобы к нему могли подключиться из сети)
netorium controller start --host 0.0.0.0 --port 8765
```
> [!IMPORTANT]
> Оставьте этот терминал открытым. Контроллер должен работать на протяжении всего тестирования.

### 2. Определение локального IP-адреса Компьютера А
Вам нужно узнать IP-адрес Компьютера А в вашей локальной сети (например, `192.168.1.15`).
* **Linux**: выполните команду `ip a` или `hostname -I`.
* **macOS**: выполните команду `ipconfig getifaddr en0`.
* **Windows**: выполните команду `ipconfig` в cmd или PowerShell.

### 3. Создание токена доступа (Enrollment Token)
Откройте **второе (новое) окно терминала** на Компьютере А и создайте токен, с помощью которого агент на другом компьютере сможет зарегистрироваться на вашем контроллере:

```bash
netorium controller token create --zone accounting --ttl 24h
```
*Скопируйте полученный токен (он начинается на `ng_enroll_...`).*

Ваш адрес для подключения агентов будет выглядеть так: `http://<IP_КОМПЬЮТЕРА_А>:8765` (например, `http://192.168.1.15:8765`).

---

## 💻 Шаг 2. Установка и подключение Агента (на Компьютере Б — "Рабочий ПК")

Компьютер Б — это управляемый ПК под управлением **Windows**.

### 1. Установка агента
На Компьютере Б откройте **PowerShell от имени Администратора** (Пуск ➡️ введите "PowerShell" ➡️ нажмите правой кнопкой мыши ➡️ выберите "Запуск от имени администратора") и выполните команду установки:

```powershell
irm https://raw.githubusercontent.com/it31wasdrexm/netorium/main/get.ps1 | iex
```

### 2. Регистрация агента на Контроллере
Свяжите агент на Компьютере Б с контроллером на Компьютере А, используя URL и токен из Шага 1:

```powershell
# Замените IP-адрес (192.168.1.15) и токен на ваши реальные данные!
netorium-agent enroll --controller http://192.168.1.15:8765 --token ng_enroll_ВАШ_ТОКЕН_ИЗ_ШАГА_1
```
Убедиться, что привязка прошла успешно, можно командой:
```powershell
netorium-agent status
```

### 3. Запуск агента в режиме ожидания команд
Для того чтобы агент начал получать и применять правила брандмауэра и блокировок, запустите его в PowerShell (также с правами Администратора):

```powershell
netorium-agent run
```
> [!IMPORTANT]
> Оставьте это окно PowerShell открытым, чтобы агент получал новые правила от контроллера в реальном времени.

---

## 🔍 Шаг 3. Проверка подключения агента на Контроллере (Компьютер А)

Вернитесь во второй терминал на Компьютере А (Админский ПК) и проверьте, видит ли контроллер наш подключенный рабочий ПК:

```bash
netorium policy agents
```
или
```bash
netorium controller agent list
```

В таблице вы увидите данные вашего агента. Обратите внимание на:
* **Agent ID** (например, `agt_a1b2c3d4`)
* **Hostname** (имя компьютера, например, `pc-accounting-01`)

> [!NOTE]
> В командах управления ниже вы можете использовать конкретное имя компьютера (например, `pc-accounting-01`), его уникальный ID (например, `agt_a1b2c3d4`) или ключевое слово `all`, чтобы применить правила ко всем агентам сразу.

## 🚫 Шаг 4. Блокировка сайтов

Блокировка сайтов выполняется агентом через автоматическое внесение записей в файл `hosts` на Компьютере Б.

### 1. Пробный запуск (Dry-run)
По умолчанию команды отправляются в безопасном тестовом режиме (без реального применения на ПК), чтобы проверить доставку:
```bash
netorium policy site all youtube.com -r "Тест блокировки YouTube"
```
На Компьютере Б в терминале запущенного агента вы должны увидеть, что команда получена и успешно имитирована.

### 2. Реальная блокировка сайта
Чтобы применить блокировку на самом деле, добавьте флаг `--real`:
```bash
netorium policy site all youtube.com -r "Реальная блокировка YouTube" --real
```
* **Проверка:** Попробуйте открыть `youtube.com` в любом браузере на Компьютере Б. Доступ должен быть заблокирован.
* *Решение проблем:* Если сайт всё равно открывается, очистите кэш DNS на Компьютере Б в командной строке (`ipconfig /flushdns`) и обновите страницу в браузере (или откройте её в режиме Инкогнито).

### 3. Снятие блокировки сайта
Чтобы разблокировать сайт обратно, используйте флаг `--unblock`:
```bash
netorium policy site all youtube.com -r "Разблокировка YouTube" --unblock --real
```

## 🎮 Шаг 5. Блокировка игр и приложений

Блокировка игр и программ на Компьютере Б происходит через создание блокирующих правил в Брандмауэре Windows (Windows Firewall) для исходящего трафика.

### 1. Блокировка игры по имени исполняемого файла (`.exe`)
Если игра установлена в стандартных папках (Steam, Program Files, LocalAppData), агент сможет найти её самостоятельно по названию процесса:

```bash
# Блокируем Counter-Strike 1.6 для всех компьютеров
netorium policy game all cs1.6.exe -r "Запрет игр в рабочее время" --real

# Блокируем Dota 2 для конкретного компьютера по Hostname
netorium policy game pc-accounting-01 dota2.exe -r "Запрет Dota 2" --real
```

### 2. Блокировка по полному пути к exe-файлу
Если игра установлена в нестандартную директорию (например, на диск `D:\`), укажите полный путь:

```bash
netorium policy app all "D:\Games\Counter-Strike 1.6\cs1.6.exe" -r "Точная блокировка по пути" --real
```

### 3. Разблокировка игры
Чтобы снять блокировку с игры или приложения, отправьте команду с флагом `--unblock`:

```bash
netorium policy game all cs1.6.exe -r "Конец рабочего дня" --unblock --real
```

## ⚡ Шаг 6. Ограничение скорости интернета (Шейпинг)

Netorium умеет урезать скорость интернета для агента с помощью службы Windows QoS (Quality of Service). Скорость задается в килобитах в секунду (Кбит/с).
*Например: `2048` Кбит/с — это 2 Мбит/с, а `512` Кбит/с — это 0.5 Мбит/с.*

### 1. Ограничение входящей и исходящей скорости
Чтобы ограничить скачивание до 2 Мбит/с, а отдачу до 512 Кбит/с для конкретного ПК:

```bash
netorium policy speed pc-accounting-01 --down 2048 --up 512 -r "Ограничение скорости сотрудника" --real
```

### 2. Ограничение только исходящей скорости для всех ПК
```bash
netorium policy speed all --up 512 -r "Ограничение отдачи" --real
```

### 3. Полное снятие ограничений скорости
Чтобы сбросить все скоростные лимиты на компьютере:

```bash
netorium policy clear-speed all -r "Сброс лимитов" --real
```

## 📋 Шаг 7. Просмотр очереди и истории команд

Вы всегда можете посмотреть список отправленных команд на контроллере:

```bash
# Показать команды для всех агентов
netorium policy list

# Показать команды только для определенного агента
netorium policy list agt_a1b2c3d4
```

## ❔ Частые проблемы и их решение

* **Команда отправлена с контроллера, но на агенте ничего не происходит:**
  * Убедитесь, что на Компьютере Б запущен процесс агента (`netorium-agent run`).
  * Убедитесь, что окно PowerShell запущено с правами Администратора.
  * Проверьте связь между ПК с помощью `ping <IP_Адрес_Компьютера_А>` на Компьютере Б.
* **Ошибка "Windows-only feature":**
  * Функции ограничения скорости (QoS) и блокировки игр (Брандмауэр) поддерживаются только на операционной системе Windows. Убедитесь, что агент работает на Windows.
* **Сайт заблокирован, но всё равно загружается в браузере:**
  * Браузеры активно кэшируют адреса или используют технологию DNS-over-HTTPS. Сбросьте кэш DNS командой `ipconfig /flushdns` на Компьютере Б, очистите кэш браузера или временно отключите опцию "Безопасный DNS" в его настройках.
